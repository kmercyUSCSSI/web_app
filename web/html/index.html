Education and Early Career

Kevin Mercy, Owner and Founder of MercyGIS LLC., is a trained GIS Analyst, GIS Developer, and Software Engineer. At the beginning of Kevin’s career at the University of Southern California, Kevin worked as a Research Associate in the University of Southern California Spatial Sciences Institute and Archaeology Research Center and learned many skills at the core of GIS Analysis and GIS Environmental Research. Kevin gained experience working with vector GIS data, raster GIS data, 3D GIS data (i.e ladar, lidar, photogrammetric images, point clouds, etc.), and worked mainly using commercial Esri ArcGIS Desktop products to perform analysis for many interdisciplinary projects. One of the main takeaways from his early career was the difficulty in working with high resolution datasets within commercial Esri ArcGIS Desktop software.

Influence of Software Engineering and Advanced Technologies

There were always many ways to accomplish geoprocessing workflows using Esri ArcGIS Desktop Products and local computing; however, the computers crashed often, or it seemed that it would take forever to compute solutions. At this point, Kevin shifted to study computer science and software engineering to learn how to leverage more advanced scripting, web, and cloud technologies to process or investigate GIS data. Through his further study and career progression, Kevin learned Python, JavaScript, HTML, CSS, and C++ and begin to work on scripting geoprocessing routines using the Esri arcpy module, and also with open-source Python libraries such as GDAL, PDAL, geopandas, and rasterio. Upon advancing his knowledge in scripting and software engineering, Kevin gained expansive skills in automation of GIS geoprocessing tasks and imagery analysis and exposure to high performance and cloud computing technologies.

Founding of MercyGIS LLC.

Through MercyGIS LLC., owner and founder Kevin Mercy provides consulting services in GIS Analysis, GIS Development, and Software Engineering. Kevin seeks to assist clients in making custom tools, scripts, or programs to run geoprocessing tools at scale, or to deploy GIS tasks in high performance computing, web, or cloud environments. Although Kevin specializes in GIS data, Kevin is also a full-stack software engineer with experience in setting up databases and designing front-end user interfaces, and a Python and C++ developer. He also works on general software engineering projects in Python, JavaScript/HTML/CSS, and C++ outside of GIS.

Services

Custom ArcGIS Pro GeoProcessing Scripting and Tool Creation

Expand your geoprocessing workflows in Esri ArcGIS Pro with custom geoprocessing tool scripting within Python, and option for configuration of scripts as reusable Esri ArcGIS Pro tools. Custom geoprocessing workflows can be converted into Python Script which is runnable within Esri ArcGIS Pro or as a Esri ArcGIS Pro toolbox tool. Custom scripts can help solve problems that are difficult within Model Builder. Iterate over large datasets and perform the same processing steps and compare. Implement custom algorithms that aren’t in Esri ArcGIS Pro.

With custom scripting, your GIS data is more useable and more powerful.

Front-End Web Design

Custom website design using JavaScript, HTML, and CSS. Most site deployments have been done using Bootstrap.js for front-end design and node.js express for server development. Front-end Web GIS development includes open source and commercial solutions (i.e Cesium JS, Esri ArcGIS JavaScript API, Esri ArcGIS Developer and Experience Builder, OpenLayers, etc.). Solutions can be hosted locally (and privately) or through cloud providers.

Website design can also be completed in Python using Flask or other http services.

Back-End Database Design, Implementation, and Administration

Database development, implementation, and administration using Python django module with connection to PostGRE SQL Database with PostGIS extension. Other databases within local or cloud networks can also be configured. In addition to database creation, custom middleware or back-end data processing functions or routes can be configured to serve data effectively to front-end resources. Most middleware development has been completed in JavaScript using node.js server to route PostGRE SQL data into front-end JavaScript webpage and using django views to create REST APIs.
